# 高校业务覆盖大屏系统设计

日期：2026-06-05

## 目标

建设一个以大屏展示为核心的小型高校业务覆盖系统。系统用于录入高校交付记录，并在前端通过地图钻取、指标汇总、标签筛选和高校详情展示业务覆盖情况。第一版优先保证演示效果、录入便利、代码复用和后续字段扩展能力。

## 已确认范围

- 技术栈采用 Next.js、React、TypeScript、Bun，后续适配 Vercel 预览部署。
- 前端是系统核心，包含大屏地图、动画交互、指标面板、标签筛选和详情抽屉。
- 后端只做轻量数据能力：记录增删改查、CSV 导入导出、数据汇总接口。
- CSV 按“一行一条交付记录”导入，同一个高校可以有多条交付记录。
- 地图钻取层级为：全国 → 省份 → 地区/城市 → 高校 → 交付资源详情。
- 管理页包含 CSV 导入/导出、数据表筛选、单条新增、编辑、删除。
- 初版字段允许不完全稳定，保留扩展字段以支持后续追加。
- 需要 mock 数据用于测试大屏和交互效果。
- 代码统一简洁、可复用，并使用必要的中文注释说明复杂逻辑。
- 项目需要创建 AGENTS.md 和 README.md 说明。

## 非目标

- 第一版不做复杂账号权限、审计流、审批流。
- 第一版不面向多人同时高频编辑同一份数据。
- 第一版不引入重型数据库，字段稳定后再考虑迁移到 Postgres 或其他数据库。
- 静态 HTML 模式不承诺服务端持久化，只提供浏览器本地数据能力。

## 运行模式

系统支持三种运行方式，同一套页面组件通过数据适配层切换数据来源。

### Vercel 预览模式

- 使用 Next.js 页面和 Route Handlers。
- API 部署为 Vercel Functions。
- 数据存储使用 Vercel Blob 保存 JSON 数据文件。
- 适合在线预览、演示和共享访问。
- 需要注意 Blob 覆盖更新可能存在短暂缓存传播延迟，因此第一版不处理多人并发写冲突。

### 本地服务模式

- 使用 `bun dev` 或 `bun start` 运行。
- API 读写本地 JSON 数据文件。
- 适合本机录入、维护数据和调试交互。
- 本地数据文件使用 `data/deliveries.local.json`，不提交到仓库。

### 静态 HTML 模式

- 使用 `bun run export` 生成 `out/` 静态目录。
- 该模式不依赖服务端 API，使用浏览器本地存储保存数据。
- 支持查看大屏、CSV 导入、CSV 导出和本地编辑。
- 静态产物是一组 HTML、CSS、JS 和资源文件，不是单个独立 HTML 文件。
- 建议通过静态 Web 服务器打开，而不是直接使用 `file://`。

## 架构

```text
UI 页面层
  ├─ 大屏总览
  ├─ 地图钻取
  ├─ 高校详情
  └─ 数据管理

业务能力层
  ├─ 数据汇总
  ├─ 标签筛选
  ├─ CSV 解析与生成
  ├─ 地图层级状态
  └─ 表格筛选与编辑

数据适配层
  ├─ API DataProvider
  └─ Browser DataProvider

存储层
  ├─ 本地 JSON
  ├─ Vercel Blob JSON
  └─ 浏览器本地存储
```

页面组件只依赖统一的 DataProvider 接口，不直接关心数据来自 API、Blob、本地 JSON 还是浏览器存储。这样可以复用大屏、管理页、CSV 和汇总代码。

## 页面设计

### 大屏总览

首屏展示整体高校覆盖情况，适合投屏和演示。页面包含：

- 全国覆盖 KPI：高校数量、省份数量、地区数量、交付记录数量、产品覆盖数量。
- 中国地图区域：默认全国视图，点击省份后进入省份放大，继续点击地区/城市查看高校列表和高校点位。
- 标签筛选：采购标签和产品标签均支持多选筛选。
- 业务分布：按产品、采购标签、项目阶段、覆盖状态展示统计。
- 高校排行：按交付数量、产品覆盖数量或最近交付排序。
- 高校详情抽屉：展示高校下所有交付资源记录。

地图交互需要支持面包屑返回，例如“全国 / 广东省 / 深圳市 / 深圳大学”。

### 数据管理

管理页以实用录入为主，不做复杂后台。页面包含：

- 交付记录表格。
- 省份、地区/城市、高校、产品标签、采购标签、项目阶段、覆盖状态筛选。
- 新增记录。
- 编辑记录。
- 删除记录。
- 批量选择和批量删除可作为低成本增强。

### 导入导出

导入导出能力包含：

- 下载 CSV 模板。
- 上传 CSV 并预览解析结果。
- 导入前展示错误和警告，例如缺少高校名称、资源数量不是数字。
- 导出当前全量数据为 CSV。
- 导出当前筛选结果为 CSV。

## 初版数据模型

CSV 一行表示一条交付记录。字段如下：

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| id | 系统生成 ID | 否 |
| province | 省份 | 是 |
| city | 地区/城市 | 是 |
| university | 高校名称 | 是 |
| longitude | 经度 | 否 |
| latitude | 纬度 | 否 |
| customerStatus | 客户状态 | 否 |
| coverageStatus | 覆盖状态 | 否 |
| projectStage | 项目阶段 | 否 |
| deliveryDate | 交付日期 | 否 |
| owner | 负责人 | 否 |
| purchaseTags | 采购标签，多个用分号分隔 | 否 |
| productTags | 产品标签，多个用分号分隔 | 否 |
| resourceType | 资源类型 | 否 |
| resourceAmount | 资源数量 | 否 |
| resourceUnit | 资源单位 | 否 |
| deliveryContent | 交付内容 | 否 |
| notes | 备注 | 否 |
| extraJson | 扩展字段 JSON | 否 |

初始采购标签包含：VMware替换、信创、AI超融合。

初始产品标签包含：SDDC、FastGPT、EDS、桌面云。

字段命名在代码内部使用英文，页面表单和 CSV 模板展示中文列名。CSV 导入时优先支持中文列名，同时兼容英文字段名。

## 数据汇总规则

- 全国覆盖数量按去重高校数量计算。
- 省份覆盖数量按 `province + university` 去重计算。
- 地区覆盖数量按 `province + city + university` 去重计算。
- 交付记录数量按记录行数计算。
- 产品覆盖按 `productTags` 拆分后统计。
- 采购标签覆盖按 `purchaseTags` 拆分后统计。
- 高校详情按同一高校聚合所有交付记录。

同名高校跨省时按省份区分。同一高校在同一地区存在多条交付记录时，地图高校数量只计一次，交付资源明细显示多条。

## 视觉与交互方向

使用项目内 `.codex/skills/ui-ux-pro-max` 生成的方向，整体为“数据密集型 Dashboard + Drill-down Analytics”。

- 视觉基调：企业级、清晰、适合大屏展示。
- 色彩：以蓝色数据色为主，使用琥珀色强调重点，不做单一深蓝或过度科幻风。
- 字体：优先使用适合数据展示的字体组合，保持数字和标签易读。
- 地图：使用分级设色地图和高校点位/气泡，不依赖只有颜色可辨的信息。
- 动效：使用 GSAP 做页面进入、钻取过渡和面板切换；Three.js 可用于背景或局部 3D 数据氛围，但不能影响主要可读性。
- 无障碍：尊重 `prefers-reduced-motion`，保留文字表格替代地图数据。

## 技术选型

- 应用框架：Next.js + React + TypeScript。
- 包管理和脚本：Bun。
- 动效：GSAP。
- 可视化：地图使用 SVG/GeoJSON 或轻量 D3 能力；图表可使用 Recharts 或自定义 SVG。
- 图标：Lucide React。
- CSV：使用稳定 CSV 解析库处理引号、逗号、换行等情况。
- 样式：优先使用 CSS Modules 或全局 CSS 变量，避免把样式逻辑散落到业务代码中。
- 数据存储：本地 JSON、Vercel Blob JSON、浏览器本地存储。

## 模块边界

- `components/dashboard/`：大屏布局、KPI、地图容器、详情抽屉。
- `components/admin/`：表格、表单、导入导出 UI。
- `components/ui/`：按钮、输入框、标签、抽屉、弹窗等基础组件。
- `lib/data/`：DataProvider 接口、API Provider、Browser Provider。
- `lib/csv/`：CSV 解析、模板生成、导出生成。
- `lib/analytics/`：汇总、筛选、去重、排行。
- `lib/map/`：地图层级、区域选择、坐标处理。
- `app/api/`：Next.js API 路由。
- `data/`：mock 数据和本地开发数据。

每个模块只暴露清晰接口。复杂汇总逻辑放在 `lib/analytics/`，保证大屏和管理页能复用同一套统计结果。

## 错误处理

- CSV 导入时对必填字段做校验。
- 对错误行展示行号、字段名和原因。
- 对缺失经纬度的高校仍允许导入，但地图点位可能只出现在列表中。
- API 写入失败时保留表单数据并提示重试。
- 静态模式下提示数据保存在当前浏览器。
- Blob 未配置时，Vercel 预览可回退到只读 mock 数据并给出配置提示。

## 测试与验收

第一版验收标准：

- `bun install`、`bun dev` 能启动项目。
- 大屏能展示 mock 数据的全国覆盖数量、省份数量、地区数量和高校数量。
- 点击全国地图省份后进入省份视图，点击地区后展示高校列表，点击高校后展示交付资源详情。
- 产品标签和采购标签筛选能影响 KPI、地图和列表。
- 管理页能新增、编辑、删除交付记录。
- CSV 模板下载、CSV 导入、CSV 导出可用。
- 静态导出模式能生成 `out/`，并支持浏览器本地数据导入导出。
- README 包含运行、构建、静态导出、Vercel 部署和数据字段说明。
- AGENTS.md 包含项目开发约定。

测试策略：

- 对 `lib/analytics/` 的汇总和筛选逻辑写单元测试。
- 对 `lib/csv/` 的解析、中文列名映射和导出写单元测试。
- 对 DataProvider 写轻量接口测试或 mock 测试。
- 开发完成后使用 Playwright 或浏览器截图检查桌面和移动主要页面，确保大屏、管理页无明显重叠。
